### テスト-実装乖離の自動検出（自動検出: 2026-03-28）

**プロジェクト**: Small World
**状況**: 3種の実装変更（モデル定数変更、リトライロジック追加、パラメータ制限変更）がテスト3ファイルを
同時に failing させた。変更箇所は小さいが影響範囲が広い。

**チェックリスト（実装変更時に必ず確認）:**

1. **定数/設定値を変更** → `grep` でテストファイル内の同じ値を検索し、同期更新
   ```bash
   # 例: モデル名を変更した場合
   grep -rn "旧モデル名" tests/
   ```

2. **関数のリトライ/フォールバックロジック追加** → モック呼び出し回数の期待値を確認
   ```javascript
   // Before: expect(mock).toHaveBeenCalledTimes(1)
   // After:  expect(mock).toHaveBeenCalledTimes(2) // リトライ分
   ```

3. **配列操作（slice/filter/map）の変更** → テスト側の期待データ長を確認
   ```javascript
   // Before: traits全件を期待
   // After:  slice(0,2) による先頭2件のみを期待
   ```

4. **外部API応答形式の変更** → モックの応答構造を同期

**根本原因**: 実装変更時にテストファイルの影響分析を怠った。テストは本番コードの「契約」であり、契約変更時は必ず両方を更新すべき。

### 本番データでのE2Eテスト禁止（自動検出: 2026-03-28）

**プロジェクト**: Small World
**状況**: 本番 Firestore のプリセットエージェント（Mia）を E2E テスト中に削除してしまった。

**ルール:**
- E2E テストは必ずエミュレータ or テストアカウントで実行
- 本番環境のデータを直接操作するテストは絶対に書かない
- テスト用データは `beforeEach` / `afterEach` で作成・削除する
