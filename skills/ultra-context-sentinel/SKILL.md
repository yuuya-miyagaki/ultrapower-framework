# Ultra Context Sentinel — セッション健全性監視

> **長時間セッションでの「忘却」「脱線」「資源枯渇」を防止する。**
> AIセッションの最大リスクは、セッション中盤以降にコンテキストが失われ、同じ作業を繰り返すか、暗黙の前提を忘れて矛盾した変更をすること。

## 起動条件

- **プロアクティブ提案（推奨）**: ultra-onboard 完了後、またはセッション開始時にエージェントが提案する
- **手動起動**: 「コンテキスト確認」「進捗確認」「予算確認」
- **定期チェック**: 大きなタスク完了ごと / 30分ごと（推定）

> **学習パターン参照**: 同ディレクトリの `learned-patterns.md` が存在する場合、スキル実行時に参照し、過去の知見を活用する。

## モード 1: セッション開始時チェック（Session Bootstrap）

### 1.1 前回セッションの引き継ぎ確認

```yaml
# Memory MCP から前回のセッション情報を取得
mcp_memory_search_nodes: "session-status"
mcp_memory_search_nodes: "[プロジェクト名]"
```

### 1.2 作業コンテキストの構築

```bash
# 現在のブランチと最新コミットを確認
run_command: git branch --show-current 2>/dev/null && git log --oneline -5 2>/dev/null

# 未完了の変更を確認
run_command: git status --short 2>/dev/null

# TODO/FIXME の確認
run_command: grep -rn "TODO\|FIXME\|HACK\|XXX" src/ --include="*.{js,ts,jsx,tsx,py}" 2>/dev/null | head -15
```

### 1.3 セッション台帳の初期化

```yaml
SESSION_LEDGER:
  session_id: "[タイムスタンプ]"
  project: "[プロジェクト名]"
  started_at: "[開始時刻]"
  goals:
    - "[目標1]"
    - "[目標2]"
  completed: []
  blocked: []
  assumptions:
    - "[前提条件1: 例) テストDBはローカルで動作]"
    - "[前提条件2: 例) nodeバージョンは18以上]"
  freeze_zones: []
  context_checkpoints: 0
```

## モード 2: 定期ヘルスチェック（Heartbeat）

> 大きなタスク完了時や、セッションが長引いている場合に実行。

### 2.1 進捗サマリー

```text
╔══════════════════════════════════════════════════╗
║  CONTEXT SENTINEL — ヘルスチェック #[N]          ║
╠══════════════════════════════════════════════════╣
║  セッション経過:  [N] 分                         ║
║  完了タスク:      [N] / [N]                      ║
║  現在のタスク:    [タスク名]                      ║
║  ブロッカー:      [あり/なし]                     ║
╠══════════════════════════════════════════════════╣
║  コンテキスト健全度:  [🟢/🟡/🔴]                ║
╚══════════════════════════════════════════════════╝
```

### 2.2 コンテキスト健全度の判定

| レベル | 条件 | 対応 |
| ------ | ---- | ---- |
| 🟢 **HEALTHY** | 全タスクの前提が有効、freeze 違反なし | 続行 |
| 🟡 **DEGRADED** | 前提条件の1つが未確認/古い、3タスク以上完了で要確認 | 前提条件の再検証を推奨 |
| 🔴 **CRITICAL** | コンテキスト喪失の兆候（同じ調査を繰り返し、矛盾する変更）| **即座に停止**して状況整理 |

### 2.3 コンテキスト喪失の兆候検出

以下のパターンを検知した場合、🔴 CRITICAL を報告:

| 兆候 | 検出方法 |
| ---- | -------- |
| **同じファイルの再読み込み** | 同一ファイルを3回以上読んでいる |
| **矛盾する変更** | 直前の変更を元に戻す変更を実施 |
| **前提の忘却** | セッション序盤に合意した設計判断と矛盾する実装 |
| **脱線** | 現在のタスクと無関係なファイルの編集 |
| **堂々巡り** | 同じエラーに3回以上遭遇（ultra-debug のエスカレーションと連携） |

### 2.4 CRITICAL 時の対応プロトコル

```text
🔴 CONTEXT SENTINEL — CRITICAL ALERT

検出された問題: [兆候の詳細]
推奨アクション:
  1. 現在の作業を中断
  2. SESSION_LEDGER の goals と completed を確認
  3. 未完了の assumptions を再検証
  4. 必要に応じてセッション引き継ぎ（モード 3）を実行
```

## モード 3: セッション引き継ぎ（Handover）

> セッション終了時、または新しいセッションへの引き継ぎ時に実行。

### 3.1 セッションサマリーの生成

```yaml
SESSION_SUMMARY:
  session_id: "[ID]"
  duration: "[N] 分"
  goals_achieved:
    - "[完了タスク1]"
    - "[完了タスク2]"
  goals_remaining:
    - "[未完了タスク1: 理由]"
  decisions_made:
    - "[設計判断1: 理由]"
    - "[設計判断2: 理由]"
  known_issues:
    - "[問題1: 状況]"
  next_steps:
    - "[次のアクション1]"
    - "[次のアクション2]"
  context_for_next_session: |
    [次回セッションの冒頭で確認すべき重要なコンテキスト]
```

### 3.2 Memory MCP への永続化

```yaml
mcp_memory_create_entities:
  - name: "session-[プロジェクト名]-[日付]-[seq]"
    entityType: "session_handover"
    observations:
      - "GOALS: [達成/未達成の概要]"
      - "DECISIONS: [重要な設計判断]"
      - "ISSUES: [既知の問題]"
      - "NEXT: [次のアクション]"
      - "CONTEXT: [引き継ぎコンテキスト]"

mcp_memory_create_relations:
  - from: "session-[プロジェクト名]-[日付]-[seq]"
    to: "[プロジェクト名]"
    relationType: "session_of"
```

### 3.3 ファイルベース永続化（Memory MCP 未使用時）

```bash
run_command: mkdir -p docs/session-handovers
```

`docs/session-handovers/YYYY-MM-DD-session-[seq].md` に保存。

## モード 4: MCP 予算モニタリング

> MCP ツール上限（100ツール等）が設定されている場合の予算管理。

### 4.1 予算ファイル確認

```bash
# mcp_budget.md の確認
run_command: cat mcp_budget.md 2>/dev/null || echo "No MCP budget file found"
```

### 4.2 予算レポート

```text
╔══════════════════════════════════════════╗
║  MCP 予算ステータス                      ║
╠══════════════════════════════════════════╣
║  総ツール上限:    [N]                    ║
║  使用済み:        [N] ([N]%)             ║
║  残余:            [N]                    ║
║  消費ペース:      [N] ツール/タスク      ║
║  推定残タスク数:  [N]                    ║
╠══════════════════════════════════════════╣
║  ステータス: [🟢/🟡/🔴]                ║
╚══════════════════════════════════════════╝
```

| レベル | 条件 | 対応 |
| ------ | ---- | ---- |
| 🟢 | 残余 50% 以上 | 通常運用 |
| 🟡 | 残余 20-50% | ツール使用を効率化。不要な search/read を削減 |
| 🔴 | 残余 20% 未満 | **即座にユーザーに報告**。最低限のツール使用に制限 |

## レッドフラグ

- ❌ コンテキスト喪失を検知しても無視して続行する
- ❌ freeze 違反を黙って修正する（ユーザーに報告すべき）
- ❌ MCP 予算が 🔴 なのに大量のツール呼び出しを続ける
- ❌ セッション終了時に引き継ぎを行わない
