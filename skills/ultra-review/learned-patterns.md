### ホットスポット駆動リファクタリング警告（自動検出: 2026-03-28）

**プロジェクト**: Small World
**状況**: `dashboard.js` が12回変更され、チャット/タスク管理/エージェント作成の3機能が密結合。
変更数上位3ファイルはリファクタリング候補として自動フラグを立てるべきだった。

**ルール:**

1. **同一ファイルへの変更が5回超** → 関心の分離（SoC）違反の可能性。分割検討を提案
2. **変更数上位3ファイル**をレビュー時に必ずチェック
3. ホットスポット検出コマンド:
   ```bash
   git log --since=<window> --format="" --name-only | sort | uniq -c | sort -rn | head -5
   ```

**アンチパターン:**
- ❌ 1ファイルに複数の関心事（UI表示 + ルーティング + 状態管理）を混在
- ❌ 「後でリファクタリング」で放置 → 変更コストが指数的に増加

**推奨アクション:**
- レビュー時にホットスポット分析を自動実行
- 変更頻度 > 5回のファイルには "🔥 Hot Module" タグを付与
- タブベースUIの場合、各タブの内容を独立モジュールに分割
