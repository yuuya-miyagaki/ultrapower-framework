# ウルトラパワーフレームワーク — はじめかたガイド

> 🔰 初心者向け。すべて Antigravity（AI）との会話で進められます。

---

## 🧠 そもそもこれは何？

**AIと一緒に開発するときの「ルールブック」** です。

| 例え | 実際のもの |
|------|-----------|
| 🍳 レシピ本 | **ウルトラパワーフレームワーク**（このリポジトリ） |
| 🥘 実際に作る料理 | **あなたのプロジェクト**（Webアプリなど） |
| 👨‍🍳 料理するたびにレシピ本にメモを足す | **振り返りでフレームワークを改善** |

---

## 📁 フォルダの全体像

```
作業フォルダ/
│
├── ultrapower-framework/     ← レシピ本（GitHub管理）
│   ├── AGENTS.md
│   ├── skills/
│   └── GUIDE.md              ← このファイル
│
├── .agents/                  ← ★AIへの接続
│   ├── AGENTS.md → リンク
│   └── skills/   → リンク
│
├── my-project/               ← プロジェクトA（GitHub管理）
└── next-project/             ← プロジェクトB（将来）
```

**大事なルール:**
- ✅ フレームワークとプロジェクトは **別々のフォルダ・別々のGitリポ**
- ❌ フレームワークの中にプロジェクトのコードを入れない

---

## 🚀 初回セットアップ（1回だけ）

### ステップ 1: Antigravity を開く

Antigravity を起動して、**「GitHubからクローン」** を選びます。

```
リポジトリURL: https://github.com/yuuya-miyagaki/ultrapower-framework.git
```

> これでフレームワークのフォルダが開かれた状態になります。

### ステップ 2: AIに作業フォルダを整えてもらう

AIにこう言います：

```
「このフレームワークを使ってプロジェクトを進めたいです。
作業フォルダを整えてください。」
```

AIが以下をやってくれます：
1. 一つ上の階層を作業フォルダとして整理
2. `.agents/` フォルダを作り、フレームワークへのシンボリックリンクを設定
3. 準備完了を報告

具体的には、AI が以下のコマンドを実行します：

```bash
# 作業フォルダに .agents/ ディレクトリを作成
mkdir -p ../.agents
# AGENTS.md と skills/ へのシンボリックリンクを設定
ln -sf ../ultrapower-framework/AGENTS.md ../.agents/AGENTS.md
ln -sf ../ultrapower-framework/skills ../.agents/skills
```

> 💡 自分でターミナルを打つ必要はありません。AIが全部やってくれます。

## 🆕 新しいプロジェクトを始める

### 方法 A: フォルダを一から作ってもらう

作業フォルダを Antigravity で開いた状態で、こう言います：

```
「my-new-app というプロジェクトを作って、TODOアプリを作りたい」
```

AIが自動で：
1. `my-new-app/` フォルダを作成
2. フレームワークのルールに従い、設計スキル（ultra-brainstorm）を起動
3. 開発を開始

### 方法 B: 既存のGitHubリポを追加する

```
「my-project リポジトリをこの作業フォルダにクローンして」
```

## 🔄 開発の流れ

全部AIとの会話だけで進みます：

```
┌─────────────────────────────────────────────┐
│                                             │
▼                                             │
① 「〇〇を作りたい」 → 設計が始まる            │
    │                                         │
    ▼                                         │
② 「計画OKです」 → 実装が始まる                │
    │                                         │
    ▼                                         │
③ 「レビューして」 → 品質チェック               │
    │                                         │
    ▼                                         │
④ 「デプロイして」 → 本番に出荷                 │
    │                                         │
    ▼                                         │
⑤ 「振り返りをしたい」 → フレームワーク改善     │
    │                                         │
    ▼                                         │
⑥ 「フレームワークにpushして」 → GitHubに保存  │
    │                                         │
    └── 次のプロジェクトへ ─────────────────────┘
```

### AIに言うだけで動くフェーズ一覧

| あなたが言うこと | AIがやること |
|-----------------|-------------|
| 「〇〇を作りたい」 | アイデアを整理して設計書を作る |
| 「計画OKです、作って」 | テストを書いてからコードを書く |
| 「レビューして」 | コード品質をチェック |
| 「テストして」 | ブラウザで動作確認 |
| 「並列で実行して」 | 複数の独立タスクを同時に実行 |
| 「デプロイして」 | 本番環境にアップロード |
| 「振り返りをしたい」 | 良かった点・改善点を分析 |
| 「フレームワークに反映して」 | フレームワークのファイルを修正 |
| 「pushして」 | GitHubに保存 |

## 🌱 フレームワークを育てる

プロジェクトが一区切りついたら：

```
あなた: 「振り返りをしたい」

AI:  📊 分析結果:
      - テストを先に書いたおかげでバグが少なかった（Good!）
      - Firebaseのセキュリティルールのテストが不足していた（改善点）

     📝 フレームワーク改善提案:
      1. 「Firebase エミュレータテスト手順」をルールに追加

あなた: 「いいね、反映して」

AI: ultrapower-framework/AGENTS.md を修正しました ✅

あなた: 「GitHubにpushして」

AI: pushしました ✅
```

こうして **プロジェクトを作るたびにフレームワークが強くなっていきます**。

## 🗂️ プロジェクトが増えたとき

作業フォルダに横に並べるだけ：

```
作業フォルダ/
├── ultrapower-framework/   ← 常に1つ（どんどん育つ）
├── .agents/                ← フレームワークへのリンク
│
├── project-alpha/            ← プロジェクト① ✅完了
├── project-beta/             ← プロジェクト② ✅完了
└── my-new-app/             ← プロジェクト③ 🔄作業中
```

新しいプロジェクトを始めるときは：

```
あなた: 「new-idea というプロジェクトを作って、〇〇を作りたい」
```

これだけです。フレームワークは既にリンクされているので、自動で適用されます。

## ❓ よくある質問

### Q: ターミナルを自分で打つ必要ある？
**A: ありません。** AIに言えば全部やってくれます。

### Q: フレームワークを更新したら、過去のプロジェクトに影響ある？
**A: ありません。** 更新は次のプロジェクトから適用されます。

### Q: プロジェクトとフレームワーク、両方pushするの？
**A: はい。別々にpushします。** でも「pushして」と言えばAIがやってくれます。

### Q: チームで使える？
**A: はい。** フレームワークのGitHubリポを共有すれば、同じルールで開発できます。
